# Модуль Stats

Модуль `stats` — это компонент системы **Explore With Me**, отвечающий за сбор и предоставление статистики по посещениям (хитам) различных эндпоинтов приложения.

## Архитектура модуля

Модуль состоит из трёх основных частей:

1. **stats-client** — клиентская библиотека для интеграции со stats‑сервером. Позволяет другим сервисам легко отправлять и запрашивать статистику.
2. **stats-dto** — модуль с DTO‑классами (Data Transfer Objects), определяющими контракты API между клиентом и сервером.
3. **stats-server** — серверная часть, реализующая бизнес‑логику: приём хитов, хранение данных и предоставление агрегированной статистики.

## Назначение и возможности

Модуль позволяет:
* сохранять информацию о каждом посещении (хите) определённого эндпоинта;
* получать агрегированную статистику по посещениям за указанный период;
* фильтровать статистику по приложениям, URI и уникальности IP‑адресов;
* учитывать уникальные просмотры (по IP).

## Основные компоненты

### StatsClient (stats-client)

`StatsClient` — основной класс клиентской библиотеки. Представляет собой Spring‑компонент (`@Component`), который использует `RestTemplate` для взаимодействия с stats‑сервером по HTTP.

**Основные методы:**

* `saveHit(EndpointHitDto hitDto)` — отправляет на сервер информацию о новом хите.
* `getStats(StatsRequestDto requestDto)` — запрашивает агрегированную статистику согласно параметрам запроса.

**Валидация данных:**
* проверка обязательных полей в `EndpointHitDto` (`app`, `uri`, `ip`, `timestamp`);
* проверка корректности формата IP‑адреса;
* валидация параметров запроса в `StatsRequestDto` (`start`, `end`, корректность списка `uris`).

**Логирование:**
* детальное логирование запросов и ответов (уровень DEBUG);
* обработка и информирование об ошибках взаимодействия с сервером.

### DTO (stats-dto)

Содержит классы‑модели для передачи данных:
* `EndpointHitDto` — описывает один хит (посещение): приложение, URI, IP‑адрес, временная метка.
* `StatsRequestDto` — задаёт параметры запроса статистики: период (`start`/`end`), список URI для фильтрации, флаг уникальности (`unique`).
* `ViewStatsDto` — представляет результат агрегированной статистики: приложение, URI, количество просмотров (всего и уникальных).

### Серверная логика (stats-server)

Реализует REST‑контроллеры для обработки запросов:
* `/hit` — приём новых хитов (метод POST);
* `/stats` — предоставление агрегированной статистики (метод GET).

Обрабатывает бизнес‑правила и сохраняет данные в хранилище (например, PostgreSQL).
